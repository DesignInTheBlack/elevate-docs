Elevate CSS features a compound property for setting the color of a resizing caret:

<br/>

### `transition` Property Modifiers

| Property   | Expected Values | Example     |
|------------|--------------------|-----------------------|
| `transition` | `ultra-fast` (all 0.05s ease-in-out)<br/>`fast` (all 0.1s ease-in-out)<br/>`medium` (all 0.2s ease-in-out)<br/>`slow` (all 0.3s ease-in-out)<br/>`ultra-slow` (all 0.5s ease-in-out)<br/>`ease` (all 0.3s ease)<br/>`ease-in` (all 0.3s ease-in)<br/>`ease-out` (all 0.3s ease-out)<br/>`linear` (all 0.3s linear)<br/>`snappy` (all 0.2s cubic-bezier(0.4, 0, 0.6, 1))<br/>`fluid` (all 0.4s cubic-bezier(0.25, 1, 0.5, 1))<br/>`relaxed` (all 0.6s ease-in-out) |  `transition:snappy`  |

<Aside title="Effects are Token Driven">
As with all other effects in Elevate CSS, transitions are considered to be a "design decision" and as such are treated as design system tokens. This means you can override the preset options, bring your own, etc
</Aside>
