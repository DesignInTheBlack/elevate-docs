---

Elevate CSS features two compound properties for setting the transform behavior of an element:

<br/>

### `transform` Property Modifiers

| Property   | Expected Values | Example     |
|------------|--------------------|-----------------------|
| `transform` | PassThroughToken (e.g.,  `(translateX(10px))` ) |  `transform:(translateX(10px))`  |
| `origin` | PassThroughToken (e.g.,  `(top_left)` ) |  `origin:(top_left)`  |

<br/>
<br/>

### Structured Transform Helpers (Pass Through)

These properties forward the value you provide directly into the CSS property:

| Property   | Expected Values | Example     |
|------------|------------------|-------------|
| `translate` | PassThroughToken (e.g., `(10px,_20px)` ) | `translate:(10px,_20px)` |
| `scale` | PassThroughToken (e.g., `(1.1)` ) | `scale:(1.1)` |
| `rotate` | PassThroughToken (e.g., `(45deg)` ) | `rotate:(45deg)` |
| `skew` | PassThroughToken (e.g., `(skew(10deg))` ) | `skew:(skew(10deg))` |

<br/>
<br/>

Note: `skew` forwards its value into the `transform` property.

<Aside title="'Pass Through' Means Literally">
When using PassThroughToken based properties or using it in your own custom syntax, remember that the actual CSS value is whatever you pass through. So, if you're using a PassThroughToken for a background image, for example, you must use the `url()` syntax just as you would in a CSS declaration. PassThroughTokens also allow you to write spaces in your utility strings by placing an underscore (_) in place of the space.
</Aside>
