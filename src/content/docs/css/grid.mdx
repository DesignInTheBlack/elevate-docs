Elevate CSS simplifies CSS Grid layouts by introducing a series of concise shorthands for defining grid properties. This guide explains how to use the `grid` property with modifiers for rows, columns, and gaps as well as setting row and column spans with the `row-spans` and `col-spans` properties.

<br/>

## `grid` Property Modifiers
Elevate supports the following modifiers for the `grid` property:

| **Modifier**             | **Expected Values**              | **Example**                          |
|--------------------------|----------------------------------|--------------------------------------|
| Rows           | `rows-2` or `rows-2-fr` (up to 12)         | `grid:rows-2`                         |
| Cols            | `cols-4` or `cols-4-fr` (up to 12)         | `grid:cols-4-fr`                      |
| Gap            | SpacingToken (e.g., `d3`)      | `grid:gap-d3`                        |
| ColGap         | SpacingToken (e.g., `d3`)      | `grid:colgap-d3`                     |
| RowGap         | SpacingToken (e.g., `d3`)      | `grid:rowgap-d3`                     |

<br/>

## Grid Areas

Use pass-through values to define template areas and assign areas to items:

| Property   | Expected Values | Example |
|------------|------------------|---------|
| `grid-template-areas` | PassThroughToken (e.g., `("header_header"_"sidebar_main")`) | `grid-template-areas:("header_header"_"sidebar_main")` |
| `grid-area` | PassThroughToken (e.g., `(header)`) | `grid-area:(header)` |

<br/>

## `row-spans` and `col-spans` Property Modifiers
The `row-spans` and `col-spans` properties can be used to specify the number of rows or columns a cell should span across. These properties accept the following syntax:

| **Modifier**             | **Expected Values**              | **Example**                          |
|--------------------------|----------------------------------|--------------------------------------|
| From     | NumericToken (e.g., `1`, up to 12)         | `col-spans:from-1:to-3`                         |
| To          | NumericToken (e.g., `3`, up to 12)         | `row-spans:from-2:to-4`                         |
| All          | None (direct property)         | `col-spans-all`                         |
| All          | None (direct property)         | `row-spans-all`                         |

Note that you can also add `fr` to the end of the `rows` and `cols` modifiers to indicate fractional rows or columns. That is, evenly distribute the available space among the specified number of rows or columns.

```html
<!-- Four Columns of Evenly Distributed Space -->
<div class="grid:cols-4-fr">
```

## `auto` Property Modifiers
The `auto` properties utilize the passthrough token to allow for more traditionally robust control over the layout of grid items. These properties accept the following syntax:

| **Modifier**             | **Expected Values**              | **Example**                          |
|--------------------------|----------------------------------|--------------------------------------|
| 'auto-col'    | PassThroughToken (e.g.,  `(minmax(100px,_auto))` )          | `auto-col:(minmax(100px,_auto))`                         |
| 'auto-row'        | PassThroughToken (e.g.,  `(minmax(20%,_80vmax))` )         | `auto-row:(minmax(20%,_80vmax))`                         |
| 'auto-flow'    | PassThroughToken (e.g.,  `(row_dense)` )         | `auto-flow:(row_dense)`                         |

<br/>
<br/>
<br/>

<Aside title="'Pass Through' Means Literally">
When using PassThroughToken based properties or using it in your own custom syntax, remember that the actual CSS value is whatever you pass through. So, if you're using a PassThroughToken for a background image, for example, you must use the `url()` syntax just as you would in a CSS declaration. PassThroughTokens also allow you to write spaces in your utility strings by placing an underscore (_) in place of the space.
</Aside>

## Examples of Grid Layouts

Elevate CSS ensures creating grid layouts is intuitive and concise, enabling dynamic design with minimal effort while simultaneously preserving the power and flexibility of CSS Grid.

```html
<!-- Declaring a Grid Layout -->
<div class="grid:gap-d3:rows-2:cols-4">
    <div class = "col-spans:from-1:to-3"></div>
</div>
```
